# 00.5 – Instalar Python no Windows + básico do PowerShell

Aqui você vai instalar o **Python** (a linguagem que faz o `app.py` funcionar) e aprender 5 comandos do **PowerShell** (o "terminal" do Windows) que vão te servir pro resto da vida.

> **Já tem Python instalado?** Pule para a seção [Conferir se já está instalado](#conferir-se-ja-esta-instalado) lá no final. Se o teste passar, vá direto para [`01-CRIAR-PROJETO-SUPABASE.md`](01-CRIAR-PROJETO-SUPABASE.md).

---

## Parte 1 – Instalando o Python

### Passo 1.1 – Baixar o instalador

1. Abra seu navegador (Chrome, Edge, Firefox)
2. Vá em **https://www.python.org/downloads/**
3. Você vai ver um botão grande amarelo escrito **"Download Python 3.x.x"** (o número exato pode mudar — qualquer versão **3.10 ou superior** serve. Em junho/2026, a versão mais nova é a 3.12 ou 3.13).
4. Clique no botão. Um arquivo `python-3.x.x-amd64.exe` vai começar a baixar.

### Passo 1.2 – Rodar o instalador

1. Vá até a pasta **Downloads** do seu computador e dê **duplo clique** no arquivo baixado.
2. Vai aparecer uma janela do instalador.

### Passo 1.3 – ATENÇÃO, o passo mais importante

Antes de clicar em "Install Now", **olhe pra parte de baixo da janela**. Tem uma caixinha de seleção escrita:

> ☐ **Add python.exe to PATH**

**MARQUE essa caixinha.** Ela tem que ficar com o ✓.

> **Por que isso é tão importante?**
> Se você não marcar, depois você vai digitar `python` no PowerShell e o Windows vai responder _"python não é reconhecido"_, como se o Python não existisse. Marcar essa caixinha é o que ensina o Windows a saber onde está o Python. Se esquecer, dá pra refazer, mas é chato — então marque agora.

### Passo 1.4 – Instalar

1. Clique em **"Install Now"** (a opção de cima, recomendada).
2. Vai aparecer um pedido de permissão do Windows ("Deseja permitir que este aplicativo faça alterações?"). Clique em **Sim**.
3. Aguarde de 1 a 3 minutos. Vai aparecer uma barra de progresso azul.
4. Quando terminar, vai aparecer a tela **"Setup was successful"**. Clique em **Close**.

Python instalado.

## Parte 2 – Abrindo o PowerShell

Agora vamos abrir o terminal do Windows. **PowerShell** é onde você digita comandos pro computador fazer coisas (como instalar bibliotecas, rodar o `app.py`, etc.).

### Como abrir:

1. Aperte a tecla **Windows** (ou clique no botão Iniciar)
2. Digite **`powershell`**
3. Vai aparecer **"Windows PowerShell"** na lista de resultados
4. Clique para abrir

Vai aparecer uma janela **azul escura** com um texto tipo:

```
PS C:\Users\SeuNome>
```

Essa janela é o PowerShell. O `PS C:\Users\SeuNome>` é o **prompt** — significa "estou pronto, pode digitar". Tudo que você digitar e apertar Enter, o computador vai tentar executar.

> **Dica:** "PS" no começo significa "Power Shell" (não "post-scriptum"). O texto depois é a **pasta atual** onde você está no momento. Sempre que abrir o PowerShell, ele começa na sua pasta de usuário.

## Parte 3 – Conferir se o Python foi instalado certo

No PowerShell, digite o comando abaixo e aperte **Enter**:

```powershell
python --version
```

Você deve ver uma resposta tipo:

```
Python 3.12.4
```

(O número pode ser diferente — qualquer 3.10 ou maior está OK.)

Se você ver isso: **deu certo, segue em frente.**

### Se aparecer "python não é reconhecido"

Você esqueceu de marcar a caixinha "Add python.exe to PATH" na hora de instalar. Soluções:

- **Opção 1 (recomendada):** Desinstale o Python (Painel de Controle → Programas → Python → Desinstalar) e instale de novo, dessa vez marcando a caixinha.
- **Opção 2 (avançado):** Veja a seção sobre PATH no [`07-PROBLEMAS-COMUNS.md`](07-PROBLEMAS-COMUNS.md).

### Também teste o pip

`pip` é o "instalador de bibliotecas" do Python. Vem junto com o Python. Teste:

```powershell
pip --version
```

Resposta esperada:

```
pip 24.x ...
```

Se aparecer um número, ótimo. Se aparecer "não reconhecido", é o mesmo problema do PATH — refaça a instalação.

## Parte 4 – Os 5 comandos do PowerShell que você precisa saber

Vou explicar 5 comandos básicos. Pode anotar num post-it.

### 1) `cd` — entrar numa pasta

`cd` é "**c**hange **d**irectory" (mudar de pasta). Use quando quer ir pra outra pasta.

```powershell
cd C:\Users\SeuNome\Projects\Projeto-Agrom-quinas
```

> **Dica de ouro:** No PowerShell, você não precisa digitar o caminho todo. Comece a digitar o nome de uma pasta e aperte **Tab** — o PowerShell completa pra você.

### 2) `ls` — listar arquivos da pasta atual

```powershell
ls
```

Mostra todos os arquivos e subpastas da pasta onde você está agora. No Windows também funciona `dir`.

### 3) `pwd` — onde estou?

```powershell
pwd
```

Mostra o caminho completo da pasta atual. Útil quando você esquece onde está.

### 4) `cls` — limpar a tela

```powershell
cls
```

Apaga tudo da janela do PowerShell e volta o cursor pro topo. Não apaga nada do computador — só "limpa a sujeira visual".

### 5) `Ctrl + C` — parar o que está rodando

Quando você roda algo que fica "preso" (como o `python app.py`, que fica ligado o tempo todo esperando requisições), aperte **Ctrl + C** no teclado pra parar.

## Parte 5 – Como navegar até a pasta do projeto

Você vai precisar disso muito. Vamos praticar.

### Achar o caminho da pasta do projeto

1. Abra o **Explorador de Arquivos** do Windows (ícone de pastinha)
2. Navegue até a pasta onde está o seu projeto `Projeto-Agrom-quinas`
3. Clique uma vez na **barra de endereço** (em cima)
4. Vai aparecer o caminho completo, tipo: `C:\Users\samuca\Documents\Projeto-Agrom-quinas`
5. Aperte **Ctrl+C** para copiar

### Navegar até lá no PowerShell

No PowerShell, digite:

```powershell
cd "COLE_AQUI_O_CAMINHO"
```

Exemplo real:

```powershell
cd "C:\Users\samuca\Documents\Projeto-Agrom-quinas"
```

> **Por que as aspas?** Se o caminho tem espaços em branco (tipo `Meu Projeto`), o PowerShell precisa das aspas pra entender que é um nome só. Se não tem espaço, não precisa, mas usar aspas sempre nunca dá erro.

Para confirmar que você chegou, digite:

```powershell
ls
```

Você deve ver os arquivos do projeto: `app.py`, `index.html`, `schema.sql`, etc. Se viu, está na pasta certa.

## Conferir se já está instalado

Se você suspeita que já tem Python (de algum curso antigo), abra o PowerShell e digite:

```powershell
python --version
pip --version
```

- Se **ambos** responderem com um número (Python 3.10+), você já tem tudo. Vá para [`01-CRIAR-PROJETO-SUPABASE.md`](01-CRIAR-PROJETO-SUPABASE.md).
- Se algum responder "não é reconhecido" ou se o Python for versão 3.9 ou menor, faça a instalação acima.
